01

Instalación Windows Server 2022

ℹ️
Verificar requisitos previos: mínimo 2 GB RAM (16 GB recomendado), 32 GB disco para SO, NIC Gigabit. Tener a mano clave de licencia y configuración de red.
🖥️
Requisitos Mínimos
  • ✓CPU: 1.4 GHz 64-bit
  • ✓RAM: 512 MB (Desktop Experience: 2 GB)
  • ✓Disco: 32 GB mínimo
  • ✓NIC Gigabit Ethernet
  • ⊙TPM 2.0 (recomendado)
📋
Ediciones Disponibles
  • ✓Standard — hasta 2 VMs
  • ✓Datacenter — VMs ilimitadas
  • ✓Essentials — hasta 25 usuarios
  • ⊙Core vs. Desktop Experience

Descargar la ISO desde el Volume Licensing Service Center (VLSC). Crear USB booteable con Rufus (GPT + UEFI). Verificar hash SHA256 con: Get-FileHash .\WS2022.iso -Algorithm SHA256

Habilitar: Secure Boot, VT-x / AMD-V (virtualización), Hyperthreading. Boot order: USB primero. Deshabilitar CSM si se usa GPT/UEFI puro.

Seleccionar "Windows Server 2022 Standard (Desktop Experience)", aceptar licencia, elegir "Custom: Install Windows only", particionar disco (100 MB EFI + 16 GB MSR + restante C:). Instalación desatendida disponible con fichero unattend.xml.

Cambiar nombre del equipo, configurar IP estática, habilitar RDP, instalar actualizaciones (Windows Update), instalar VMware Tools / Guest Additions, configurar NTP y unir al dominio si procede.

bash — post-install setup
#!/bin/bash # Script: ubuntu-post-install.sh # Propósito: Configuración inicial servidor Ubuntu 24.04 # Ejecutar como root o con sudo # Actualizar sistema apt update && apt upgrade -y # Instalar herramientas esenciales apt install -y \ vim curl wget git htop net-tools \ openssh-server ufw fail2ban \ sysstat nmap traceroute # Configurar hostname hostnamectl set-hostname srv-linux-01 echo "192.168.30.20 srv-linux-01.empresa.local srv-linux-01" >> /etc/hosts # Hardening SSH (/etc/ssh/sshd_config) sed -i 's/#PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config sed -i 's/#PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config sed -i 's/X11Forwarding yes/X11Forwarding no/' /etc/ssh/sshd_config # Configurar UFW (firewall) ufw default deny incoming ufw default allow outgoing ufw allow from 192.168.10.0/24 to any port 22 ufw allow 80/tcp ufw allow 443/tcp ufw --force enable # Crear usuario admin local useradd -m -s /bin/bash itadmin usermod -aG sudo itadmin echo "itadmin ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers.d/itadmin systemctl restart ssh echo "[OK] Configuración inicial completada"

Biblioteca de Scripts PowerShell

✅
Todos los scripts requieren política de ejecución RemoteSigned. Ejecutar primero: Set-ExecutionPolicy RemoteSigned -Scope LocalMachine
powershell — gestión de active directory
# ============================================ # Script: AD-UserManagement.ps1 # Descripción: Gestión masiva de usuarios AD # Requiere: Módulo RSAT-AD-PowerShell # ============================================ # Importar módulo AD Import-Module ActiveDirectory # ── Crear usuario desde CSV ────────────────── function New-BulkADUsers { param([string]$CsvPath) $usuarios = Import-Csv $CsvPath foreach ($u in $usuarios) { $password = ConvertTo-SecureString "P@ssw0rd2026!" -AsPlainText -Force New-ADUser ` -Name "$($u.Apellido), $($u.Nombre)" ` -GivenName $u.Nombre ` -Surname $u.Apellido ` -SamAccountName $u.Usuario ` -UserPrincipalName "$($u.Usuario)@empresa.local" ` -Path "OU=Usuarios,DC=empresa,DC=local" ` -Department $u.Departamento ` -AccountPassword $password ` -ChangePasswordAtLogon $true ` -Enabled $true Write-Host "[+] Creado: $($u.Usuario)" -ForegroundColor Green } } # ── Reporte de usuarios inactivos ──────────── function Get-InactiveUsers { param([int]$DaysInactive = 90) $cutoff = (Get-Date).AddDays(-$DaysInactive) Get-ADUser -Filter { LastLogonDate -lt $cutoff -and Enabled -eq $true } ` -Properties LastLogonDate, Department, Manager | Select-Object Name, SamAccountName, Department, LastLogonDate | Sort-Object LastLogonDate | Export-Csv "C:\IT\Reports\InactiveUsers_$(Get-Date -f 'yyyyMMdd').csv" -NoTypeInformation Write-Host "Reporte generado." -ForegroundColor Cyan } # ── Bloquear usuario comprometido ──────────── function Disable-CompromisedUser { param([string]$Username) Disable-ADAccount -Identity $Username Clear-ADAccountExpiration -Identity $Username Get-ADUser $Username | Move-ADObject -TargetPath "OU=Bloqueados,DC=empresa,DC=local" Write-Warning "Usuario $Username BLOQUEADO y movido a OU Bloqueados." } # === Uso === # New-BulkADUsers -CsvPath "C:\IT\nuevos_usuarios.csv" # Get-InactiveUsers -DaysInactive 60 # Disable-CompromisedUser -Username "jhacker"
