01

Protocolos de Firewall

🔴
Cualquier modificación a reglas del firewall perimetral debe ser aprobada por el responsable de seguridad y documentada con ticket en el sistema ITSM antes de su implementación.
📋 Política General (Default Deny)
🔴 Política Perimetral — Inbound

Todo el tráfico entrante está DENEGADO por defecto. Solo se permiten los puertos explícitamente autorizados mediante reglas numeradas.

🟢 Política Perimetral — Outbound

El tráfico saliente desde la red interna está PERMITIDO por defecto, con excepciones para protocolos de riesgo (Telnet, FTP sin cifrar).

🟡 Política DMZ — Bidireccional

La DMZ usa reglas explícitas en ambas direcciones. Los servidores en DMZ NO tienen acceso directo a la red interna; todo acceso va por el firewall.

🔵 Gestión (VLAN 10)

Acceso SSH/RDP a dispositivos de red solo desde IPs de la VLAN de administración (192.168.10.0/24). Autenticación doble factor obligatoria.

cisco ASA — ACL ejemplo
! === ACL PERIMETRAL - CISCO ASA 9.x === ! Permitir HTTPS entrante a servidor web en DMZ access-list OUTSIDE_IN extended permit tcp any host 10.10.40.10 eq 443 access-list OUTSIDE_IN extended permit tcp any host 10.10.40.10 eq 80 ! SSH solo desde red de administración access-list INSIDE_IN extended permit tcp 192.168.10.0 255.255.255.0 any eq 22 ! Denegar telnet en todos los sentidos access-list OUTSIDE_IN extended deny tcp any any eq 23 access-list INSIDE_IN extended deny tcp any any eq 23 ! Aplicar ACL a interfaces access-group OUTSIDE_IN in interface outside access-group INSIDE_IN in interface inside ! Verificar show access-list OUTSIDE_IN show conn

Protocolo de Respuesta a Incidentes

Escala de Severidad
CRÍTICO P1
Brecha de datos / Ransomware / Sistema caído en producción
Respuesta: INMEDIATA · Escalado: CIO + Responsable seguridad · SLA: <15 min
ALTO P2
Acceso no autorizado detectado / Servicio crítico degradado
Respuesta: <1 hora · Escalado: Responsable área · SLA: <4 horas resolución
MEDIO P3
Malware aislado / Cuenta comprometida sin propagación
Respuesta: <4 horas · Escalado: Técnico senior · SLA: 24 horas
BAJO P4
Actividad sospechosa / Fallo menor sin impacto
Respuesta: Día laboral · Escalado: Técnico disponible · SLA: 72 horas
powershell — auditoría de eventos de seguridad
# ============================================ # Script: Get-SecurityAudit.ps1 # Descripción: Auditoría de eventos críticos AD # ============================================ param( [int]$HorasAtras = 24, [string]$Servidor = "localhost" ) $desde = (Get-Date).AddHours(-$HorasAtras) # Eventos de seguridad críticos a monitorizar: # 4624 - Inicio de sesión exitoso # 4625 - Inicio de sesión fallido # 4648 - Logon con credenciales explícitas # 4720 - Cuenta de usuario creada # 4726 - Cuenta de usuario eliminada # 4768 - Solicitud de ticket Kerberos (TGT) $eventIDs = 4625, 4648, 4720, 4726 Get-WinEvent -ComputerName $Servidor -FilterHashtable @{ LogName = 'Security' Id = $eventIDs StartTime = $desde } | ForEach-Object { $xml = [xml]$_.ToXml() [PSCustomObject]@{ Hora = $_.TimeCreated EventID = $_.Id Usuario = $xml.Event.EventData.Data | Where { $_.Name -eq 'TargetUserName' } | Select-Object -Exp '#text' Equipo = $_.MachineName Resultado = switch($_.Id) { 4625 { "LOGIN FALLIDO" } 4720 { "CUENTA CREADA" } 4726 { "CUENTA ELIMINADA" } default { "EVENTO: $_" } } } } | Sort-Object Hora -Descending | Export-Csv "C:\IT\Security\Audit_$(Get-Date -f 'yyyyMMdd_HHmm').csv" -NoTypeInformation Write-Host "[OK] Auditoría completada." -ForegroundColor Green

powershell — hardening checklist
# ============================================ # Script: Invoke-HardeningCheck.ps1 # Descripción: Verificación de seguridad básica # ============================================ function Test-SecurityBaseline { $checks = @() # 1. Firewall de Windows activo en todos los perfiles $fw = Get-NetFirewallProfile $checks += [PSCustomObject]@{ Control = "Windows Firewall" Estado = if (($fw | Where {-not $_.Enabled}).Count -eq 0) { "OK" } else { "FALLO" } } # 2. Windows Defender activo $av = Get-MpComputerStatus $checks += [PSCustomObject]@{ Control = "Windows Defender" Estado = if ($av.AntivirusEnabled) { "OK" } else { "FALLO" } } # 3. Actualizaciones pendientes $updates = (Get-WindowsUpdate -ErrorAction SilentlyContinue).Count $checks += [PSCustomObject]@{ Control = "Windows Updates" Estado = if ($updates -eq 0) { "OK" } else { "PENDIENTES: $updates" } } # 4. Cuenta de administrador local deshabilitada $admin = Get-LocalUser "Administrator" $checks += [PSCustomObject]@{ Control = "Local Admin deshabilitado" Estado = if (-not $admin.Enabled) { "OK" } else { "REVISAR" } } # Mostrar resultado $checks | Format-Table -AutoSize $failed = $checks | Where { $_.Estado -ne "OK" } if ($failed) { Write-Warning "$($failed.Count) controles fallidos. Revisar urgente." } else { Write-Host "[OK] Baseline de seguridad superado." -ForegroundColor Green } } Test-SecurityBaseline
